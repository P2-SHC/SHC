## 1. 라우팅 구조

### 구현 방식
URL을 변경하지 않는 **State 기반 SPA**. `App.jsx`가 현재 페이지와 파라미터를 `useState`로 관리한다.

```js
// App.jsx 상태 목록
const [page, setPage] = useState("MainPage");
const [category, setCategory] = useState("");
const [selectedPostId, setSelectedPostId] = useState(null);
const [productId, setProductId] = useState("");
const [from, setFrom] = useState("");
const [fromPostId, setFromPostId] = useState(null);
const [healthSavedState, setHealthSavedState] = useState(null);
const [orderItems, setOrderItems] = useState([]);   // 결제 페이지 전달용
const [fromCart, setFromCart] = useState(false);    // 장바구니 출처 여부

const navigate = (pageName, params = {}) => {
  setPage(pageName);
  setSelectedPostId(params.postId ?? null);
  setCategory(params.category ?? "");
  setProductId(params.productId ?? "");
  setFrom(params.from ?? "");
  setFromPostId(params.fromPostId ?? null);
  if (params.orderItems !== undefined) setOrderItems(params.orderItems);
  if (params.fromCart !== undefined) setFromCart(params.fromCart);
};
```

### `from` 파라미터를 이용한 돌아가기 분기

`ProductDetailPage`의 돌아가기 버튼은 `from` 값에 따라 목적지를 결정한다.

| from | 이동 대상 |
|------|----------|
| `"BoardDetailPage"` | 이전 게시글 (fromPostId 사용) |
| `"MainPage"` | 메인 페이지 |
| `"HealthRecommendPage"` | AI 추천 (상태 복구) |
| 기본 | 상품 목록 |

## 2. 인증 시스템

### 비밀번호 해싱
Web Crypto API의 SHA-256을 사용한다 (별도 라이브러리 불필요).

```js
const hashPassword = async (password) => {
  const encoder = new TextEncoder();
  const data = encoder.encode(password);
  const hashBuffer = await crypto.subtle.digest('SHA-256', data);
  const hashArray = Array.from(new Uint8Array(hashBuffer));
  return hashArray.map(b => b.toString(16).padStart(2, '0')).join('');
};
```

### 사용자 저장 구조 (localStorage: `shc_users`)

```json
[
  {
    "username": "user1",
    "name": "홍길동",
    "passwordHash": "sha256해시값",
    "age": 65,
    "healthInterests": ["고혈압", "수면"]
  }
]
```

### 로그인 흐름
1. 입력 비밀번호 SHA-256 해싱
2. `shc_users`에서 username + passwordHash 일치 확인
3. 일치 시 `shc_current_user`에 저장

사용자별 독립 장바구니 키를 사용한다.



### ProductContext
```
초기화: localStorage(shc_products) → 없으면 product.json 사용
이후: setProducts() 호출마다 localStorage 동기화
```

- `decreaseStock(id, qty)`: 재고 차감 (functional updater 사용)

### AlarmContext
사용자별 약·혈당·혈압 알람 데이터를 관리한다.

```js

`shc_medicines_{username}`

`shc_blood_pressure_{username}`
```

로그인/로그아웃 시 `username` 변경을 감지해 해당 유저의 데이터를 로드한다.

```js

  name: string,
```

```js
const getWeatherKeyword = (icon) => {
  if (!icon) return '환절기';
  const code = icon.slice(0, 2);
  if (code === '01') return '자외선';
  if (['09', '10', '11'].includes(code)) return '비';
  if (code === '13') return '눈';
  return '환절기';
};
```

상품 선정:
```js
// 미세먼지 1개
const microDustPool = products.filter(p => p.keyword.includes('미세먼지'));
const microDustProduct = microDustPool.length > 0
  ? microDustPool[Math.floor(Math.random() * microDustPool.length)]
  : products[Math.floor(Math.random() * products.length)]; // 없으면 전체 랜덤

// 날씨 2개 (미세먼지 상품 제외, 부족하면 fallback)
const weatherPool = products
  .filter(p => p.keyword.includes(weatherKeyword) && p.id !== microDustProduct.id)
  .sort(() => 0.5 - Math.random());
const fallbackPool = products
  .filter(p => !p.keyword.includes(weatherKeyword) && p.id !== microDustProduct.id)
  .sort(() => 0.5 - Math.random());
const weatherProducts = [...weatherPool, ...fallbackPool].slice(0, 2);
```

## 5. 추천 알고리즘

### 5.1 메인 페이지 인기글
날짜 근접도 + 조회수를 조합한 점수로 상위 1개를 선정한다.

```js
const featured = [...articles].sort((a, b) => {
  const dateScore = new Date(b.createdAt) - new Date(a.createdAt);
  const viewScore = (b.viewCount - a.viewCount) * 86400000;
  return dateScore + viewScore;
})[0];
```

### 5.2 게시글 상세 상품 추천
카테고리 우선순위 + 키워드 교집합 점수를 조합한다.

```js
const withScore = products.map(product => ({
  ...product,
  matchCount: product.keyword.filter(k => post.keyword.includes(k)).length,
}));

if (post.category === 'life') {
  // 카테고리 무관, matchCount 내림차순
  recommendedProducts = withScore
    .filter(p => p.matchCount > 0)
    .sort((a, b) => b.matchCount - a.matchCount)
    .slice(0, 3);
} else {
  // 같은 카테고리 상품 우선, 나머지로 보완
  const preferred = withScore
    .filter(p => p.category === post.category && p.matchCount > 0)
    .sort((a, b) => b.matchCount - a.matchCount);
  const others = withScore
    .filter(p => p.category !== post.category && p.matchCount > 0)
    .sort((a, b) => b.matchCount - a.matchCount);
  recommendedProducts = [...preferred, ...others].slice(0, 3);
}
```

### 사이드바 Sticky 구현
우측 사이드바는 CSS `position: sticky`로 스크롤 시 화면에 고정된다. `align-self: start`가 없으면 Grid 셀이 메인 영역 높이만큼 늘어나 sticky가 동작하지 않으므로 반드시 필요하다.

```css
.co-sidebar {
  position: sticky;
  top: 6.5rem;    /* 헤더 높이 확보 */
  align-self: start;
}
```

모바일(780px 이하)에서는 sticky를 해제하고 사이드바를 메인 위로 올린다.

```css
@media (max-width: 780px) {

  .co-sidebar { order: -1; position: static; }
  .co-pay-btn--mobile { display: block; }  /* 메인 하단 별도 결제 버튼 노출 */
}
```

### 재고 처리 방식
`decreaseStock` 반환값에 의존하지 않고 `getStock`으로 선검증 후 일괄 차감한다.
(React 18 배칭 환경에서 `setProducts` 콜백 내 클로저 변수가 비동기 처리될 수 있기 때문)

```js
// 1단계: 전체 재고 선검증
for (const item of orderItems) {
  const stock = getStock(item.id);
  if (stock < item.quantity) {
    alert(`${item.title}의 재고가 부족합니다. (현재 재고: ${stock}개)`);
    return;
  }
}

// 2단계: 전체 통과 후 일괄 차감
orderItems.forEach(item => decreaseStock(item.id, item.quantity));

// 3단계: 장바구니 출처면 항목 제거
if (fromCart) orderItems.forEach(item => removeFromCart(item.id));
```

## 8. 건강 알리미 시스템

### AlarmNotifier 동작 원리
앱 최상위에 항상 렌더링되어 30초마다 알람 시간을 체크한다.

```js
useEffect(() => {
  if (!currentUser) return;
  checkAlarms();
  const interval = setInterval(checkAlarms, 30_000);
  return () => clearInterval(interval);
}, [currentUser?.username]);
```

### 알람 중복 방지 (firedRef)
같은 날 같은 알람이 두 번 발생하지 않도록 localStorage에 발화 기록을 저장한다.

```js
const fireKey = `med_${med.id}_${time}`;
if (time === currentTime && !firedRef.current.has(fireKey)) {
  firedRef.current.add(fireKey);
  newAlarms.push({ ... });
}

// localStorage 저장 키: shc_alarm_fired_{username}_{YYYY-MM-DD}
```

### 알람 큐 처리
동시에 여러 알람이 발생하면 `pendingAlarms` 배열에 쌓아 순서대로 표시한다.

```js
// 현재 표시: pendingAlarms[0]
// 확인(dismiss) 시 배열에서 제거 → 다음 알람 표시
```

### 약 복용 완료 처리
약 알람 확인 버튼 클릭 시 `decreaseMedicineCount(medicineId)` 호출로 잔여량 자동 차감.

```js
const dismiss = (alarm) => {
  if (alarm.type === 'medicine' && alarm.medicineId) {
    decreaseMedicineCount(alarm.medicineId); // 잔여량 -1
  }
  setPendingAlarms(prev => prev.filter(a => a.id !== alarm.id));
};
```

### 알람 팝업 메시지 구성
현재 표시 중인 알람(큐의 첫 번째)만 모달로 렌더링한다. 대기 중인 알람이 더 있으면 개수를 함께 표시한다.

```jsx
const current = pendingAlarms[0];
const icons = { medicine: '💊', blood_sugar: '🩸', blood_pressure: '❤️' };

// 대기 알람 수 표시
{pendingAlarms.length > 1 && (
  <p className="alarm-modal__more">+{pendingAlarms.length - 1}개 알림 대기 중</p>
)}
```

### Ref를 통한 최신 상태 참조
`setInterval` 콜백에서 stale closure 문제를 방지하기 위해 세 종류 알람 모두 Ref로 관리한다.

```js
const medicinesRef = useRef(medicines);
const bloodSugarRef = useRef(bloodSugarAlarms);
const bloodPressureRef = useRef(bloodPressureAlarms);

useEffect(() => { medicinesRef.current = medicines; }, [medicines]);
useEffect(() => { bloodSugarRef.current = bloodSugarAlarms; }, [bloodSugarAlarms]);
useEffect(() => { bloodPressureRef.current = bloodPressureAlarms; }, [bloodPressureAlarms]);
// checkAlarms에서 각 Ref.current 참조 → 항상 최신 상태
```

## 11. 재고 관리

### 초기화 흐름
```
앱 최초 실행
  → localStorage(shc_products) 확인
  → 없으면 product.json을 초기값으로 저장
  → 있으면 localStorage 값 사용 (이전 차감 이력 유지)
```

### decreaseStock 구현 주의사항
`setProducts` 내 클로저 변수로 성공 여부를 반환하는 방식은 React 18 배칭 환경에서 신뢰성이 낮다. **결제 흐름에서는 `getStock`으로 선검증 후 `decreaseStock`을 호출하는 방식을 사용한다.**

```js
// ❌ 신뢰성 낮음 (배칭 환경)
const success = decreaseStock(id, qty);
if (!success) { ... }

// ✅ 올바른 방식
const stock = getStock(id);
if (stock < qty) { alert(...); return; }
decreaseStock(id, qty); // 검증 후 차감
```

### 저재고 기준
- 재고 0: 품절 (구매 버튼 비활성화)
- 재고 1~5: 저재고 경고 표시

## 12. 무한 스크롤

`IntersectionObserver`로 목록 끝 요소를 감지해 5개씩 추가 로드한다.

```js
const [visibleCount, setVisibleCount] = useState(5);
const loaderRef = useRef(null);

useEffect(() => {
  const observer = new IntersectionObserver(([entry]) => {
    if (entry.isIntersecting) setVisibleCount(prev => prev + 5);
  });
  if (loaderRef.current) observer.observe(loaderRef.current);
  return () => observer.disconnect();
}, [filtered]); // 검색어/카테고리 변경 시 재등록

// JSX
<div ref={loaderRef} />
```

검색어·카테고리 변경 시 `visibleCount`를 5로 초기화하고 스크롤 위치를 상단으로 리셋한다.

## 13. localStorage 키 목록

모든 키는 `shc_` 프리픽스를 공유한다. 사용자별 데이터는 `{username}`을 포함해 계정 간 격리한다.

### 고정 키 (전역)

| 키 | 관리 주체 | 저장 시점 | 형식 |
|----|----------|----------|------|
| `shc_users` | UserContext | 회원가입 | `User[]` |
| `shc_current_user` | UserContext | 로그인 / 로그아웃 | `User` (passwordHash 제외) |
| `shc_products` | ProductContext | 초기화 / 재고 변경 | `Product[]` |
| `shc_font_scale` | FontSizeContext | 폰트 크기 변경 | `"small" \| "default" \| "large"` |

```ts
// User 구조 (shc_users 원소, shc_current_user)
{
  username: string,
  name: string,
  passwordHash: string,   // shc_current_user에는 저장하지 않음
  age?: number,
  healthInterests?: string[]
}

// Product 구조 (shc_products 원소)
{
  id: number, category: string, title: string,
  keyword: string[], description: string,
  price: number, quantity: number, image: string
}
```

### 사용자별 키 (`{username}` 포함)

| 키 패턴 | 관리 주체 | 저장 시점 | 형식 |
|---------|----------|----------|------|
| `shc_cart_{username}` | CartContext | 장바구니 추가 / 수정 / 삭제 | `CartItem[]` |
| `shc_medicines_{username}` | AlarmContext | 약 추가 / 수정 / 삭제 / 복용 차감 | `Medicine[]` |
| `shc_blood_sugar_{username}` | AlarmContext | 혈당 알람 추가 / 수정 / 삭제 | `TimeAlarm[]` |
| `shc_blood_pressure_{username}` | AlarmContext | 혈압 알람 추가 / 수정 / 삭제 | `TimeAlarm[]` |
| `shc_alarm_fired_{username}_{YYYY-MM-DD}` | AlarmNotifier | 알람 발화 시 | `string[]` (fireKey 집합) |

```ts
// CartItem 구조 (shc_cart_* 원소)
{ ...Product, quantity: number }   // quantity는 장바구니 수량 (재고 아님)

// Medicine 구조 (shc_medicines_* 원소)
{
  id: string,
  name: string,
  dosage: string,
  alarmTimes: string[],         // ["08:00", "20:00"]
  totalCount: number | null,
  remainingCount: number | null
}

// TimeAlarm 구조 (shc_blood_sugar_*, shc_blood_pressure_* 원소)
{ id: string, time: string }    // time: "HH:MM"

// fireKey 패턴
// 약:      "med_{medicineId}_{HH:MM}"
// 혈당:    "blood_sugar_{alarmId}_{HH:MM}"
// 혈압:    "blood_pressure_{alarmId}_{HH:MM}"
```

### 생명주기 정책

| 상황 | 동작 |
|------|------|
| 로그인 | `shc_current_user` 갱신, 해당 유저의 cart · alarm 데이터 로드 |
| 로그아웃 | `shc_current_user` 삭제, cart · alarm 메모리 상태 초기화 (localStorage 데이터는 보존) |
| 재고 차감 | `shc_products` 즉시 갱신 |
| 날짜 변경 | `shc_alarm_fired_*` 키가 날짜별로 생성되므로 자동으로 이전 날 기록과 분리 |
