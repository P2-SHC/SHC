# 시니어 헬스케어 기능 복구 로드맵

현재 정적으로 고정된 UI에 생명력을 불어넣기 위해 구현해야 할 기능 목록입니다.

## 1. 공통 및 인프라 (App & Header)
- **라우팅 시스템 복구**: `App.jsx`에서 `page` 상태에 따라 각 페이지 컴포넌트를 조건부 렌더링하도록 수정
- **전역 상태 관리**: 로그인 여부(`isLoggedIn`), 장바구니 담긴 개수(`cartCount`) 등을 전역적으로 관리
- **GNB 네비게이션**: 헤더의 메뉴 클릭 시 해당 페이지로 이동하는 링크 기능 연결

## 2. 메인 페이지 (MainPage)
- **동적 데이터 바인딩**: 최신 게시글 및 추천 상품 리스트를 Mock 데이터에서 가져와 출력
- **위젯 연동**: 날씨 및 미세먼지 API(또는 Mock 데이터)를 위젯에 연결하여 실시간 정보 표시
- **이미지 슬라이더/배너**: 메인 건강 이미지 영역을 실제 이미지나 슬라이더로 교체

## 3. 게시판 영역 (Board List & Detail)
- **카테고리 필터링**: 레시피, 라이프, 운동 카테고리에 맞는 게시글만 필터링하여 표시
- **검색 기능**: 제목 키워드로 게시글을 실시간 검색
- **TOP 5 랭킹**: 조회수가 높은 게시글 5개를 자동으로 집계하여 사이드바에 표시

## 4. 쇼핑 영역 (Product List & Detail)
- **상품 무한 스크롤/더보기**: '상품 더보기' 버튼 클릭 시 추가 상품 리스트 로드
- **수량 선택**: 상세 페이지에서 `+`, `-` 버튼으로 구매 수량 조절 및 총액 계산
- **장바구니 담기**: 담기 버튼 클릭 시 전역 장바구니 상태에 추가 및 성공 피드백(텍스트 변경 등) 제공

## 5. 장바구니 페이지 (CartPage)
- **아이템 관리**: 장바구니에 담긴 상품 리스트 출력, 수량 변경, 삭제 기능
- **선택 주문**: 체크박스로 선택한 상품들의 합계를 실시간 계산
- **주문하기**: 최종 금액 확인 및 주문 프로세스(알림창 등) 구현

## 6. 인증 (Login & Register)
- **로그인 폼**: 아이디/비밀번호 입력 시 유효성 검사 및 로그인 처리
- **회원가입**: 필수 정보 입력 및 '건강 관심사' 다중 선택 로직 구현
- **로그아웃**: 세션/로컬스토리지 정리 및 메인 페이지로 이동
